Git Bash Folder Navigation Cheat Sheet

1. Open Git Bash

Start by opening the Git Bash application. You can do this by searching for Git Bash in your Windows start menu and clicking on the app.

2. Check Current Directory

pwd (Print Working Directory): Displays the current directory path.

3. List Files and Folders

ls: Lists all files and directories in the current directory.

ls -a: Lists all items, including hidden files (those starting with a dot).

ls -l: Lists items in long format, showing permissions, number of links, owner, group, size, and timestamp.

ls -la: Combines the options to list all files in long format, including hidden ones.

4. Change Directory

cd directory_name: Changes the current directory to directory_name.

cd ..: Moves up one directory level.

cd: Changes to the home directory.

cd /: Changes to the root directory.

cd ~: Also changes to the home directory.

cd -: Changes to the previous directory.

5. Create a New Directory

mkdir directory_name: Creates a new directory named directory_name.

6. Delete a Directory

rmdir directory_name: Deletes an empty directory named directory_name.

rm -r directory_name: Recursively deletes a directory named directory_name and all its contents.

7. Help and Documentation

man command: Shows the manual page for a command, providing detailed information about its usage and options.

Tips for Efficient Navigation

Tab Completion: Use the Tab key to auto-complete directory names. This saves time and reduces typos.

History Navigation: Use the Up and Down arrow keys to navigate through previously entered commands.

Current Directory Shortcut: The dot (.) symbol represents the current directory, and two dots (..) represent the parent directory.
